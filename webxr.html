Web XR?

XR stands for Extended Reality wich intended to include both AR and VR and all other imersive technlogie such using mobile devices with the positional tracking or fixed display with the ablity to look around. WebXR is a new API standard for web developers to create VR or AR application with out having to deal directly with each hardware. this means users can use a web brouser to start viewing VR/AR contecnt with out having to intall extrea plugins or software. developers can also write with code once and it will work with all devices from diffrent brands such as Oculus, vive and many more.



WebXR was introduced in 2018 and is now being inclued in chrome verison 79 and WebVR was also removed based on MDN, the key caopabilities of WebXR are following but they are not limited to them. recognized compatible VR/AR devices. when useing a head set it will usually run at 90 FPS they render a 3D scene to the devices at the appropriate frame rate. mirror output 2D display such as monitors. most if not all of the applications will be using webGL as a 3D rendering engine and web audio API for sound. this will be rendered into html canvas.
